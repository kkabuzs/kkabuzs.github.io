GPU 架构详解

一，CPU 和 GPU 的区别

1.1 CPU 特点

特点：

  • 每个core很强。
  • 擅长串行

1.2 GPU 特点

特点：

  • 几千个小核心。
  • 例如 NVIDIA A100 大约 6912 CUDA Cores 。虽然GPU Core 很弱，但是并行能力很强，6912 个 Cores 同时负责计算。

总结：

  • CPU 单个核心功能复杂、控制能力强，适合低延迟、分支多、串行依赖强的任务。
  • GPU 单个执行单元相对简单，但数量多、吞吐量高，适合数据并行、规则重复、计算密集的任务。

二，SM（Streaming Multiprocessor）

介绍：
SM 是 GPU 内部的流式处理器。这个单元里面包含：

  • CUDA核心（CUDA Core）
  • 寄存器（Register）
  • 共享内存（Shared Memory）
  • Warp 调度器（Warp Scheduler）
  • 张量核心（Tensor Core）
  • 加载/存储单元（Load Store Unit (LSU)）
  • 特殊函数单元（Special Function Unit (SFU)）

举例：一个 GPU 有 80 个 SM，每个 SM 有 128 CUDA Cores。
所以：80 * 128 = 10240 CUDA Core
真正调度线程的是： SM

一个 Block 被分配给一个 SM，由 SM 负责内部的线程调度，GPU 调度任务的基本单位是 Block，而非单个线程。

补充：硬件真正执行指令的最小单位是 Warp，Block 是软件逻辑到硬件执行的任务封装。

2.1 CUDA Core

介绍：
SM 是 GPU 的核心计算单元，Block 会被分配到 SM 上拆分为 Warp 执行。CUDA Core 就是 SM 内部真正执行通用算术运算的最小硬件单元。

真正控制CUDA Core 工作的是Warp调度器。

2.3 Warp

介绍：
GPU 不是一个一个的执行 Thread ，而是一次执行 32 个 Thread 。这 32 个 Thread 就叫 Warp。
（NVIDIA GPU 硬件就是这样设计的 32 个 Thread 是一个Warp ）

所以 256 Threads = 8 Warp

Warp Scheduler 一次发射32 个 Thread。

2.4 Warp Divergence（分支发散）

GPU 在运行时，必须先执行if，在执行else。不能像CPU那样去判断跳过。所以 CUDA 尽量避免 if，switch，复杂判断。

2.5 Memory Hierarchy（内存层次）

GPU 有很多种 Memory。

  • Global Memory（全局内存）
  • Shared Memory（共享内存）
  • Register（寄存器）

CUDA 调优 百分之90都在优化减少访问 Global Memory。

2.5.1 Shared Memory（共享内存）的存在意义

例如 100 个线程都读取 A[0] 这块数据。就要访问100次 Global Memory，速度很慢。

这样只需要访问一次Global Memory，就大大加快了运行速度。

总结：
Shared Memory就是一个Block 内所有线程共享的小缓存（Scratchpad）。

  • 若要使用Shared Memory，需要用代码显式地让某些线程把数据搬到 Shared Memory。

2.6 Register（寄存器）

介绍：
速度最快，每个 Thread 都有自己的 Register，通常变量都会存在寄存器中，但是数量有限。例如 Register 的数量为65536。

  • 如果一个 Kernel 用了 太多的Register，GPU 同时运行的 Warp 数量就会下降（Occupancy 降低）。

2.6.1 Occupancy（占用率）

介绍：
Occupancy（占用率）是 CUDA 架构中，衡量单个 SM（流式多处理器）硬件线程资源被利用饱满程度的核心指标。

通俗解释：

  • GPU 希望 SM 一直运行，一个 SM 最多运行 64 Warp。但是如果 Kernel 只能发射 16 Warp ，就会导致很多硬件闲置。

结论：

  • Occupancy越高，通常越容易隐藏内存访问延迟。

注意：

  • Occupancy 高 ≠ 性能一定最好。 有些计算密集型 Kernel 即使 Occupancy 不高，也可能已经把计算单元跑满了，因此真正目标是平衡资源使用与执行效率。

2.7 Tensor Core

介绍：
专为AI的矩阵运算而设计的张量芯片。

2.8 总结

  • Warp Scheduler： 决定 GPU 如何调度线程。
  • Register： 决定线程的局部变量存放位置，也影响 Occupancy。
  • Shared Memory： CUDA Kernel 优化最重要的手段之一。
  • Tensor Core： AI 训练和推理性能的关键硬件。
  • Load Store Unit： 决定数据搬运效率，是内存带宽优化的重要组成部分。
  • CUDA Core： 执行通用计算。
  • Special Function Unit： 在涉及复杂数学函数时才会成为关注重点。

2.9 概念补充

  • VRAM（显存）： GPU 可使用的主要设备内存。独立 GPU 通常使用 GDDR 或 HBM；集成 GPU通常与 CPU 共享系统内存。
  • GDDR： 常见的图形显存，成本较低，广泛用于消费级和部分专业 GPU。
  • HBM： 高带宽显存，通常与 GPU 封装在一起，带宽高但成本也高，常用于 AI/HPC 加速器。
  • PCIe： CPU 与独立 GPU 之间主要的命令和数据传输通道，也可以支持 GPU 间通信。
  • NVLink： NVIDIA 的高速互连技术，可用于 GPU 与 GPU，以及某些平台中的 CPU 与 GPU 通信。

解释1：AI 和 HPC 计算通常需要更高的显存带宽，因此高端GPU常采用 HBM。

  • RTX4090：约 1008 GB/s
  • A100：约 2039 GB/s
  • H100：约 3 TB/s+

解释2：

  • 若没有NVLink：GPU0 –> PCIe –> CPU –> PCIe –> GPU1
  • 有NVLink：GPU0 –> NVLink –> GPU1
