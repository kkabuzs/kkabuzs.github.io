NVIDIA GPU Operator：Kubernetes GPU 自动化管理

一，为什么需要 GPU Operator

如果不用 GPU Operator，要让 Kubernetes 使用 NVIDIA GPU，需要手动完成：

GPU 节点数量较多时，手动部署容易出现：

  • 驱动版本不一致
  • 容器运行时配置错误
  • Device Plugin 遗漏
  • 监控组件版本不统一
  • 新增节点需要重复操作

GPU Operator 使用 Kubernetes Operator 模式，将这些组件自动部署到 GPU 节点，并持续检查其状态。

二，整体架构

GPU Operator 负责部署并管理 GPU 节点所需的软件组件。

五，NVIDIA Container Toolkit

Container Toolkit 负责让容器访问宿主机 GPU：

GPU Operator 可以自动：

  • 部署 NVIDIA Container Toolkit
  • 配置 containerd 或 CRI-O
  • 配置 NVIDIA 容器运行时
  • 让容器运行时支持 GPU 设备注入

注意：Driver 让宿主机识别 GPU；Container Toolkit 让容器能够使用 GPU。

六，Device Plugin

6.1 Device Plugin 是什么

NVIDIA Device Plugin 是 NVIDIA GPU 与 Kubernetes kubelet 之间的接口。

主要负责：

  1. 发现节点上的 GPU
  2. 向 kubelet 注册 GPU 资源
  3. 上报健康状态
  4. 在 Pod 启动时分配 GPU
  5. 向容器运行时提供设备分配信息

7. DCGM

7.1 DCGM 是什么

DCGM = Data Center GPU Manager
它是 NVIDIA 的数据中心 GPU 管理与诊断组件。

DCGM 可以获取：

  • GPU 利用率
  • 显存利用率
  • 温度
  • 功耗
  • 时钟频率
  • PCIe 信息
  • NVLink 状态
  • ECC 错误
  • Xid 错误
  • GPU 健康状态

DCGM 是数据中心 GPU 的监控、健康检查和诊断基础设施。

7.2 DCGM 与 DCGM Exporter

  • DCGM：从 GPU 驱动和硬件中获取指标、诊断信息。
  • DCGM Exporter：将 DCGM 指标转换成 Prometheus 格式并暴露 HTTP 接口。

九，MIG Manager

对于支持 MIG 的 GPU，GPU Operator 还可以部署 MIG Manager。
MIG 可以把一张物理 GPU 划分成多个相对隔离的实例：

用途：

  • 小模型推理
  • 多租户
  • 提高 GPU 利用率
  • 提供更细粒度的资源分配

注意：

  • 只有部分数据中心 GPU 支持 MIG
  • 修改 MIG 配置可能影响节点上正在运行的工作负载
  • MIG 配置应配合节点维护、驱逐和调度策略执行
