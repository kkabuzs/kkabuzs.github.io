NVIDIA Container Toolkit 原理

一，为什么需要 NVIDIA Container Toolkit

普通 Docker 容器默认只能使用：

  • CPU
  • 内存
  • 磁盘
  • 网络

NVIDIA Container Toolkit 的作用:

让 Docker、containerd、CRI-O 等容器运行时能够安全地把宿主机 NVIDIA GPU 设备和必要的驱动库提供给容器。

二，GPU 透传

说明：
容器继续使用宿主机的 NVIDIA 内核驱动，NVIDIA Container Toolkit 将 GPU 设备节点和所需的驱动用户态库暴露给容器。

2.1 整体链路

容器中通常包含：

  • PyTorch
  • CUDA Runtime
  • cuDNN
  • NCCL

宿主机负责提供：

  • 物理 GPU
  • NVIDIA 内核驱动
  • GPU 设备节点
  • NVIDIA Container Toolkit

2.2 --gpus all 发生了什么

2.2.1 Docker 创建 GPU Device Request

--gpus all 代表这个容器需要使用宿主机上的全部 GPU。

Docker 将这个请求传递给 NVIDIA 容器运行时相关组件。

2.2.2 NVIDIA Container Toolkit 发现 GPU

Toolkit 通过宿主机驱动识别：

  • GPU 数量
  • GPU UUID
  • GPU 设备节点
  • MIG 实例
  • 驱动版本
  • 所需驱动库

2.2.3 修改容器运行配置

在容器启动前，Toolkit 会准备 OCI 容器配置，将需要的设备和库暴露到容器中。

设备访问权限还会受到容器运行时和 cgroup 的控制。

2.2.5 容器访问 GPU

容器中的 CUDA 程序通过 libcuda.so 发出请求，最终经过设备节点和宿主机内核驱动操作物理 GPU。

不做额外隔离，同一张 GPU 可以被多个容器共同使用。
