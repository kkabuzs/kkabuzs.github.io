Transformer

一，溯源，从翻译开始

若要翻译一句话：
货拉拉拉不拉拉布拉多 ，can Huolala carry a Labrador
关键在于要理解这句话，才能翻译。
计算机很难理解，一个拉字不同的位置代表了不同的含义。

  • 拉不拉：拉可以翻译为：拉货,carry 、拉扯,drag/pull
  • 所以必须结合前面的货拉拉和后面的拉布拉多才能理解拉不拉的含义。

二，Token与向量

计算机只认识0或1这样的数字，所以在翻译之前，计算机会先将文本转化为数字。最简单的办法是将文本的每个字转化为对应的数字编号。

弊端：

  • 编号只是一个纯数字，不能承载任何语义信息。
  • 两个数字编号之间，无任何语义关联。

需要一个既可以数字化，又可以承载语义信息的办法

  • 将一句话切分为多个词块

  • 含义相近的词块会在坐标系中相互靠近。

  • 随着纬度的增多，所有已知词汇的token，都能在这个多维坐标系里找到自己的位置。

  • 词和词之间还可以被数学计算。例如：国王-男人+女人无限接近于王后
  • 拉不拉这种模糊定位的词语，结合货拉拉和拉布拉多才会被拉向正确的语义位置。

每个坐标纬度都可表示为数字，所以每个token都可以用一组数字表示。这种表示方式就叫向量（vector）

名词解释：

  • token： 每个字/词块都是一个token
  • embedding（嵌入）：模型里面有个巨大的表，真是纬度可能是几千维。把token ID映射成向量的过程就叫embedding。
  • vector（向量）： 每个token会被翻译成ID数字，这些ID数字的纬度坐标就叫向量。

一句话不仅有词义，还有顺序。若顺序颠倒，意思就会完全不同。所以模型还会为token增加一个位置信息。所以向量的内容包含：是什么和在什么位置。

三，QKV

在注意力计算中，第一步是将编码器的每个输入向量（即词的特征表示）通过线性变换映射为三个新的向量：查询向量（Query），键向量（Key）和值向量（Value）。简称Q、K、V向量。

3.1 举例说明

货拉拉拉不拉拉布拉多 除了表面意思，还有背后的意思。例如拉布拉多除了字面意思，还有宠物，狗等含义。所以可以把一句话抽象成一个词块数据库。

  • key：就是词块本身
  • value：就是词块背后的含义。

当计算机在理解拉不拉这个词块的含义时，就会拿着拉不拉在词块数据库遍历一遍，找到和拉不拉意义最大的词。

在词块数据库中进行遍历查询时，会有三个概念：查询，键，值。
但是直接拿token的向量去做这件事不够灵活，因为一个向量要扮演上面三个角色，所以需要三个权重矩阵：w_q，w_k，w_v。让每个token的向量变成三个专门的向量。
（三个权重矩阵是模型初始化时随机生成）。

  • 承担查询角色的Q向量：代表token关注的信息。
  • 承担键角色的K向量：代表token提供的信息。
  • 承担值角色的V向量：代表token背后的真正含义。

点积： Q1 * K1 的数值越大，相关程度越高。说明这个token越值得被重点关注。

最后将这些关注程度做一次softmax ，将它们转换成为一组权重。所有权重 （关注度）只和为1。

  • softmax ： 注意力分配比例公式。

使用上面公式得出的权重 加权 所有token的V向量，得出一个新向量，记为H2。

  • 新向量H2：token 拉不拉 融合了上下文信息之后的表示。

总结：

  • Q*V ：计算关注谁，关注程度。
  • √d_k ：scale防止点积数值过大，避免softmax饱和，梯度消失，使模型训练更稳定。
  • softmax：把关注程度转换为注意力权重。再乘以V，提取融合具体的语义内容。
  • 多头注意力：词与词之间的关系是多样的，有些是语义的关联，有些是语法结构的依赖，所以模型会多次不同侧重点的注意力计算。

3.2 KV cache

原理：
KV Cache 是将 Transformer Decoder 在生成第 t 个 token 时，之前所有 token 的 Key 和 Value 矩阵的计算结果缓存起来，避免在生成后续 token 时重复计算，从而大幅降低推理延时。

特点：

  • KV cache 应用于推理阶段。（K和V的值是不变的）
  • KV cache 只存在于 Decoder 解码器之中。
  • 目的是为了加速Q、K、V的两次矩阵相乘时的速度。
  • KV cache 会加大显存占用。

未使用KV cache

使用KV cache

通俗解释：

  • 不使用kv cache： 当模型推理计算时

    • 第一次输入中，经过计算，输出token1
    • 第二次输入中华，经过计算，输出token1，token2
    • 第三次输入中华人，经过计算，输出token1，token2，token3
    • 模型再预测人这个字的时候，跟上一轮的华的kv部分相关度最高，中这部分的计算就是多余的。

  • 使用kv cache： 当模型推理计算时

  • 第一次输入中，缓存中的 kv 并计算，输出token1

  • 第二次输入华，计算华的 kv 并拼接缓存 ，输出token2

  • 第三次输入人，计算人的 kv 并拼接缓存 ，输出token3

  • …. 类推

说明： Transformer架构对token的位置信息不敏感，必须通过位置编码来让模型确定每个token在句子中的位置，才能正常理解语义。在使用kv cache 缓存时，每次输入都是单个token，所以需要修改token的位置编码。通过查询kv cache，可以确认这个token前面有几个token，来确认这个token的正确位置编码。

补充：

  1. KV Cache 缓存的是全部历史
  • KV Cache 是一个累积缓冲区，保存了从开始到现在所有 token 的 K 和 V
  • 每一步都要从缓存中拼接历史 K/V 与新 K/V
  1. 每层 Transformer 都有一个独立的 KV Cache
  • 不是只有一个 KV Cache，而是 模型层数 × 注意力头数 个
  • 例如 LLaMA-7B（32层，32头）：有 1024 个独立的 KV Cache 缓冲区
